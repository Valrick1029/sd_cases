Классификация подходов к валидации:

1. Offline Evaluation (Pre-deployment)
Проводится на статических датасетах перед релизом.

*   Golden Dataset: Вручную размеченный набор запросов и эталонных ответов.
*   Автоматизированные метрики:
    *   *Текстовые совпадения:* BLEU, ROUGE (ограниченная применимость для генерации).
    *   *Семантическое сходство:* Cosine similarity эмбеддингов ответа и эталона.
    *   *LLM-as-a-Judge:* Использование сильной модели для оценки ответов слабой по критериям (релевантность, полнота, стиль).
    *   *Task-specific:* Accuracy, F1-score, Pass@k (для кода), SQL execution success rate.
*   Adversarial Testing: Проверка на устойчивость к prompt injection, jailbreaks, edge cases.

2. Online Evaluation (Post-deployment)
Оценка в производственной среде на реальных пользователях.

*   A/B Testing: Сравнение версий модели или промптов на сегментах трафика.
*   Implicit Feedback: Анализ поведения пользователя (copy/paste, edit distance, время взаимодействия, thumbs up/down).
*   Explicit Feedback: Формы обратной связи, репортинг ошибок.
*   Guardrails Monitoring: Отслеживание срабатывания фильтров безопасности (токсичность, PII leakage).

3. Системные метрики (SLA/SLO)
Валидация инфраструктуры, а не только качества генерации.

*   Latency: Time to First Token (TTFT), Total Generation Time.
*   Throughput: Requests per second, Tokens per second.
*   Cost: Cost per request, Token usage efficiency.
*   Availability: Error rates (5xx, rate limits, timeouts).

4. Архитектура Evaluation Pipeline
Компоненты системы для непрерывной валидации.

*   Evaluation Harness: Изолированный сервис для прогона тестовых сценариев.
*   Versioning: Версионирование моделей, промптов и датасетов (Model Registry).
*   CI/CD Integration: Блокировка деплоя при регрессии ключевых метрик.
*   Feedback Loop: Автоматическое включение проблемных кейсов в тренировочный набор (Fine-tuning data collection).

5. Риски и компромиссы

| Метод | Преимущество | Недостаток |
|-------|--------------|------------|
| LLM-as-a-Judge | Масштабируемость, семантическое понимание | Стоимость, потенциальный bias судьи |
| Human Evaluation | Высокая точность, золотой стандарт | Дороговизна, низкая скорость |
| A/B Testing | Реальное влияние на бизнес | Риск негативного опыта пользователей |
| Автоматические метрики | Скорость, дешевизна | Низкая корреляция с человеческим восприятием |

Рекомендация:
Внедрить многоуровневую систему. Offline-валидация на Golden Dataset для блокировки регрессий в CI. Online-мониторинг бизнес-метрик и сэмплирование запросов для периодической Human Evaluation. Обязательное логирование вход/выход для постфактум анализа инцидентов.
