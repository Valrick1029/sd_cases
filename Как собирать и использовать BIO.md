Существует множество видов источников информации о человеке:
1. Соцсети (группы, посты, друзья, география)
2. Сайты организаций (где работал, учился)
3. Патентные организации, научные журналы (патенты, публикации, выступления)
4. Различные ведомственные ресурсы (реестр судебных решений, список разыскиваемых преступников и пр.)

При сборе данных необходимо учитывать их релевантность. Следует отделять более важную информацию (с высоким влиянием на предсказания) от менее важной (с низким влиянием на предсказания).

Можно искать через различные API (Linkedin, WebScrapper, SerpAPI и другие)

Тип данных может различаться:
1. Структурированные (образование, опыт, позиции)
2. Категориальные (сфера занятости, роль в компании)
3. Количественные (также опыт, публикации)
4. Текстовые (описания позиций, достижений)
5. Графовые (связи с другими людьми или организациями)

Задачи могут быть такими:
1. Нанимать или не нанимать человека
2. Credit risk score
3. Поиск похожих людей

Решать их можно несколькими способами:
1. Классификация
2. Ранжирование
3. Matching
4. Выявление аномалий

В зависимости от типов данных в датасете можно реализовать разные подходы к обработке данных:
1. Для категориальных one-hot encoding
2. Для текстовых описаний очистка, эмбеддинги
3. Для количественных нормализация
4. Для графовых эмбеддинги графов

Важно также следить за тем, чтобы при прогнозировании не использовались факты о чело

Условимся, что данные мультимодальные, то есть отдельно нужно обрабатывать, к примеру, текстовые и количественные данные. Архитектуры можно использовать такие:

Для классификации:
1. Текст через encoder, количественные через нормализацию. Потом конкатенация в один вектор
2. Затем градиентный бустинг (LightGBM)

Для ранжирования:
 1. Так же сконкатенировать в один вектор все типы данных
 2. Затем в тот же градиентный бустинг, только получаем топ-n наиболее вероятных кандидатов

Для matching:
1. То же для связки текстовые+количественные + еще отдельный вектор под запрос
2. Потом в FAISS и через косинусную близость отбираем топ-n кандидатов

Метрики:
Классификация -- Recall, F1, ROC-AUC
Ранжирование -- Recall из топ-n, MAP
Matching -- Recall из топ-n, F1, ROC-AUC
