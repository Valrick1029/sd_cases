Предполагается, что задача заключается в поиске информации о каком-либо объекте в нескольких таблицах, дальнейшем агрегировании результата поиска и выдаче широкой сводки об объекте. Также предположим, что ответ должен поступать как можно быстрее, нагрузка на RAG большая (десятки запросов в секунду).

Последовательность следующая:
	Сбор данных в чанки из всех таблиц для каждого клиента --- Получение эмбеддингов текста всех документов --- Индексирование
Просто добавлять новый объект в базу документов

При инференсе можно так:
	Запрос --- Поиск подходящего документа --- Объединение запроса и результата поиска в промпт --- Передача промпта LLM --- Выдача LLM сводки

Можно использовать такую архитектуру:
1. Парсинг из таблиц с (sqlglot)
2. Сборка документа из описаний объекта в разных таблицах
3. Векторизация (BERT)
4. Индексация документов (FAISS), также для точности можно добавить индексацию по имени (BM25)
5. Поиск документов (и объединение, если есть еще и индексация по имени)
6. Объединение вектора запроса и вектора поиска в промпт и передача LLM (GPT)
7. Формирование LLM сводки

Для LLM лучше задать низкую температуру, так как она модель может додумывать факты при недостатке информации.
